나에게 DevOps란

  • Hani Brown

DevOps는 소프트웨어의 개발과 운영의 합성어입니다.
그러나 현재의 DevOps는 단순히 개발과 운영의 통합을 의미하는 것은 아닙니다.
애매할 수 있지만 둘다 한다는 것이 아닌 개발과 운영의 경계를 허물고 통합한다고 할 수 있습니다.
소프트웨어는 지속적으로 업데이트와 새로운 것이 생기기 때문에 빠른 속도에 못따라가고 그에 따른 효율성 즉 개발과 운영이 분리되어 생기는 병목을 없애고자 하는 역할인 것입니다.
DevOps강의 듣고 선생님께서 5가지 단어로 표현해주었을 때 가장 와닿았습니다. 

 
 문화,  자동화, 측정, 공유, 축적


DevOps는 하나의 문화이고 자동화를 추구하며 꾸준히 측정하여 지속적으로 개선해 나가는 것,
또한 그 측정된 바탕으로 팀원과 함께 투명성있게 공유하여 발전해 나아가는 것,
그리고 이 앞서 말한 것들을 기록하여 축적하여 자산을 만들어 나가는 것이라 생각합니다.
즉 제게 있어 DevOps Engineer는 올바른 문화와 대인관계를 위해 서비스 혹은 소프트웨어 라이프사이클에서 반복적인 일들을 자동화 하고 팀원간의 차이를 기술적으로 해소시키는 일을 담당합니다.

2

Boldness

적응하며 전진해야 됩니다.

Creativity

Timeline

2019.10 Swift Basic Code

저의 첫 프로그래밍 언어입니다.  Swift를 통하여 프로그래밍 언어의 대한 이해와 라이프사이클, 객체와 함수, 클래스 등 프로그램동작에 대한 이해를 하게된 계기가 되었습니다. 

2020.2 Engineer Information Processing

정보처리기사를 준비를 2월에 하였지만 코로나로 인해 8월에 시험친 걸로 기억합니다. 하지만 그동안 준비하면서 지치기도 하였지만 개발과 서버에 대한 기초적인 지식을 많이 쌓는 계기가 되었습니다. 현재 필기는 합격했으며 실기 보류중입니다. 

2020.5 HTML + CSS

본격적으로 꿈을 실현하기 위해서 자바 웹/앱 개발자과정을 시작으로 첫 수업으로 기본적인 마크업 언어를 배웠습니다. 웹에 대한 이해와 구성 및 동작을 알게된 계기가 되었습니다. 

2020.5 Git + GitHub + C

git, github는 원래 교육과정상 뒤에 배울 내용들이 었으나 제가 먼저 선생님께 건의하여 팀원을 떠나서 학생들의 편의와 원할한 버전관리를 위해 먼저 배우게 되었습니다. 또한 C언어를 배움으로써 컴퓨터와 프로그래밍 원리 이해에 대하여 발판이 되었습니다.  

2020.6 CentOS

유닉스, 리눅스 말로만 들었지 기존에 Mac os, Window를 사용해보았스나 이 계기를 통해 실질적으로 알게되었고 처음으로 다루어보았습니다. 조금더 운영체제의 원리와 서버라는 개념을 이해하는 발판이 되었습니다.

2020.7 Java

앞에서 Swift와 C언어 배움을 통해 프로그래밍 언어의 원리와 이해를 하고 Java라는 언어를 직관적으로 이해하고 습득하였습니다.  코드를 통해 JVM과 메모리, 라이프싸이클, 디버깅, 객체의 이해 등 배움에 어려움은 있었으나 많은 것을 알게된 계기가 되었습니다. 

2020.8 DBMS + SQL(MYSQL)

데이터베이스 시스템에서 자료를 처리하는 용도에 대해 알게 되었고 사용되는 구조적 데이터 질의 언어를 배우게되었습니다. 즉 이 수업의 계기로 관계형 데이터베이스 관리 시스템의 데이터를 관리하는 방법을 알게 되었고 정규화와 비정규화 기법을 통해 데이터베이스를 최적화하는 방법을 알게되었습니다.  

2020.8 Ubuntu + Network

실질적으로 DevOps의 길을 들어서게된 계기입니다. 앞에서 Centos를 배웠지만 의존성의 문제 즉 RPM으로 해결하기가 너무 어려웠고 처음으로 리눅스를 배운 것이라 쉽게 이해가 안되었지만 Ubuntu를 통해 조금더 친밀하게 그리고 리눅스 구조에 대해 더욱 알게된 계기가 되었습니다. 정보처리기사에서도 배웠지만 본격적으로 서버에 대해 배움으로써 네트워크의 구조와 원리 그리고 여러 개념들을 알게 되었습니다. 

2020.9 Docker + Docker Swarm

컨테이너의 이해와 컨테이너 기반의 오픈소스 가상화 플랫폼에 대해 공부하게 되었습니다. 즉 온프레미스에서 VMware를 통해 클러스터 구축해보고 가상화와 컨테이너 기술을 이해하고 학습하게 된 계기가 되었습니다. 

2020.9 Kubernetes + Istio +           Prometheus(Grafana)

kubernetes를 통해서 많은것을 배우고 많은 것을 느껴 여태 공부한 것 중에 제일 의미있고 흥미진지한 분야였습니다. Monolith service, micro services에 개념을 학습 하였고 온프레미스와 GKE에서 설치 및 구축하는 방법 그리고 다양한 오픈소스 컨테이너를 통해 Pod와 Deployment 구축과 롤링업데이트를 학습하였으며 무엇보다 Service mesh Istio활용과 Ingress 구축을 통해 고가용성에 대한 중요성을 크게 느꼇습니다. 또한 Helm 차트, Grafana, Prometheus 구축을 통해 모니터링에 관해 학습하였습니다. 

2020.10        Cloud Platform (AWS,GCP)

앞에서 조금씩 다뤄보았으나 본격적으로 학습을 하였습니다. 그리고 클라우드 컴퓨팅의 이해와 활용을 중점으로 공부하였으며 AWS에서 주최하는 세미나를 통해 AWS의 서비스들에 대해 조금더 알게된 계기가 되었습니다. 또한 Kubernetes 학습을 위해 GCP의 GKE를 활용해보고 공부를 하였습니다. 

2020.10 ~   Python + Opensource 
+ Portfolio 

포트폴리오 제작함으로서 앞에서 배운 것들을 복습과 조금더 이해가 되는 계기가 되었습니다. 또한 각각의 오픈소스에 대해 조금더 다양한 옵션들을 공부하였고 활용하는 방법 더 많이 알게된 계기가 되었습니다. 무엇보다 팀들과 함께 협업이 무엇인지 진정 느끼게 되었고 자동화를 위해 Python을 배우는 계기가 되었습니다.
